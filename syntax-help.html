详细语法帮助

1. 基本语法

默认顺序 语法格式 说明
1. 规则名定义 #规则名 定义规则名称（必须，以#开头）
2. 注释内容 //注释内容 添加规则注释（可选，以//开头）
3. 局部集合定义 集合名=={元素1,元素2,...} 定义局部集合，示例：BL=={bl,br,cl,cr...}（可选）
4. 匹配模式 :具体规则 定义匹配模式（必须，以:开头）
5. 排序规则 @排序规则 定义排序方式（可选，以@开头）

2. 集合相关

2.1 集合定义

语法格式：集合名=={元素1,元素2,...}

2.1.1 命名规则：

  • 单字母集合名必须大写。（如：C、V、L）
  • 多字母集合名必须以字母开头，可包含字母和数字，且大小写敏感。（如：BL、Vowels）

2.1.2 元素规则：

  • 使用逗号,分隔集合中的元素。
  • 逗号前的空位置表示空元素。（如：{,a}、{a,,b}表示包含空元素）
  • 集合元素大小写敏感。

2.2 集合分类

2.2.1 全局集合

可以在具体规则中直接引用，无需另行定义。

A. 默认全局集合（无法删除）：

  • C=={b,c,d,f,g,h,j,k,l,m,n,p,q,r,s,t,v,w,x,y,z} - 辅音字母
  • V=={a,e,i,o,u} - 元音字母
  • L=={a,b,c,d,e,f,g,h,i,j,k,l,m,n,o,p,q,r,s,t,u,v,w,x,y,z} - 所有字母

B. 自定义全局集合（可以删除）：

可在"全局集合定义"功能中更新全局集合。

2.2.2 局部集合

仅在当前规则中可用，必须先定义后使用。

限制：

  • 无法跨规则引用局部集合。
  • 当局部集合和全局集合同名时，局部集合会覆盖全局集合。

2.3 集合运算符

符号 名称 描述 示例
== 别名定义 给集合定义一个别名 X==V（定义X为V的别名，即X=={a,e,i,o,u}）
>> 集合减法 从集合中去掉另一个集合的内容 Y==L>>C（从L中去掉C的内容，结果Y=={a,e,i,o,u}）
<< 集合加法 向集合中加入另一个集合的内容 Z==C<<V（向C中加入V的内容，结果Z=={a,b,c,d,e,f,g,h,i,j,k,l,m,n,o,p,q,r,s,t,u,v,w,x,y,z}）

2.4 字面字符

使用双引号" "表示字面字符，用于匹配具体的字符或字符串。（如"e"、"tion"）

限制：

  • 字面字符大小写敏感。

3. 具体规则

3.1 基础规则

本质：基于规则拼接而成的用于匹配的字符串集合。

格式：必须以:开头，由位置标志（锚点）、集合组合而成。（如：:\bLV表示匹配前缀是任意字母+元音的单词）

特殊情况：:后为空时表示匹配全部单词，但必须保留:符号。

3.1.1 位置标志

用于指定匹配时的位置锚点：

  • \b - 单词开头
  • \e - 单词结尾
  • \-b - 不在开头
  • \-e - 不在结尾

3.1.2 可引用的集合

  • 全局集合（如：C、V、L）
  • 局部集合（当前规则中定义的）
  • 字面字符（如："e"、"tion"）
注意：多字母集合名必须用()引用；引用集合时大小写敏感。

3.1.3 集合内元素匹配优先级

  • 优先尝试非空字符串匹配。
  • 最后尝试空字符串匹配。

3.1.4 集合量词

  • 可以用+表示"一个或多个"相同的集合。（如：CV+C等同于CVC、CVVC、CVVVC...简写）
  • 小技巧：可以通过和集合定义配合，实现复杂匹配。（如：xL==L<<{,}\n:\bC(xL)+"tion"\e表示匹配辅音开头，中间可以有任意多个字母，最后以tion结尾的单词）

3.2 组合规则

格式：以::开头，由基础规则（规则名）、运算符组合而成。（如：::A||B表示匹配满足A规则或B规则的单词）

限制：

  • 规则名必须完全匹配。
  • 只能引用已有的基础规则。
  • 不允许引用已有的组合规则，即不允许嵌套。
  • 相关运算符只能用于组合规则，不能用于基础规则。

3.2.1 运算符

符号 名称 操作数 优先级 结合律 描述
~ 取反 一元 4 右 对规则结果取反
! 差集 二元 3 左 满足前者但不满足后者
&& 与 二元 2 左 同时满足两个条件
|| 或 二元 1 左 满足任一条件
注意：! 是二元差集运算符，并不是编程语言中逻辑非概念，不支持!A写法。

3.2.2 优先级与结合律

  • 优先级从高到低：~、!、&&、||（如：A||B&&C!~D 会被解析为A||(B&&(C!(~D)))）
  • ~是右结合。（如：~A~B会被解析为~(A~B)，而不是(~A)~B）
  • !、&&、||是左结合。（如：A&&B&&C会被解析为(A&&B)&&C）
  • 可以使用()改变运算顺序。如：
    • ~A&&B会先对A取反，然后与B进行与运算。
    • ~(A&&B)会先计算A&&B，然后对整个结果取反。

4. 排序规则

4.1 基础排序

格式：无@符号、@后为空或@后为逆序标志-（即@-）

排序方式：

  • 将筛选结果按首字母分组。
  • 分组按字母正序或逆序（@-）排列。
  • 组内按字母正序或逆序（@-）排列。

4.2 位置匹配排序

基于位置匹配符的灵活排序策略。支持前缀、后缀、任意位置、严格中间等多种匹配方式。

语法格式：@(排序元素)(排序元素)...

限制：

  • 最多支持三级排序，即3个排序元素。

4.2.1 排序元素

格式：每个排序元素由逆序标志、集合名和位置匹配符组合而成。

  • 单字母集合可以直接引用。（如：V或(V)）
  • 多字母集合名必须用括号引用。（如：(BL)）
  • 可选的逆序标志-放在集合名前。（如：-(BL)或(-BL)，括号内外都可）

示例：-A^，表示从筛选结果中，匹配前缀（^）为A集合中元素的单词，按集合A中的元素逆序（-）分组排列。

4.2.2 位置匹配符

符号 名称 描述 示例
^ 前缀匹配 元素必须出现在单词开头 V^匹配以元音开头
$ 后缀匹配 元素必须出现在单词结尾 C$匹配以辅音结尾
* 任意位置 元素可以出现在单词任意位置 V*匹配包含元音
~ 严格中间 元素必须出现在单词中间（不在首尾） V~匹配中间包含元音但首尾不是元音
无标识符 默认前缀 等同于^ V等同于V^

限制：

  • 可选的位置匹配符放在集合名后。（如：V^或(BL^)）
  • 集合引用如果有括号，匹配符必须放在括号内。（如：(V^)）
注意：严格中间位置匹配符~与组合规则中取反运算符~的区别。

4.2.3 多级排序

顺序关系：多个排序元素之间隐含前后关系，前一个元素在后一个元素之前，且不要求紧邻。

匹配条件：单词必须同时满足所有级别的匹配条件才能被放到对应各级分组，未分组的单词会放到"其他"分组。

示例：@(BL^)(V*)(C$)

  1. 第一级：以 BL 集合元素开头
  2. 第二级：包含 V 集合元素
  3. 第三级：以 C 集合元素结尾
  4. 单词必须同时满足：以BL开头 AND 包含V AND 以C结尾，才能正确分组。

4.2.4 分组标签

标签格式：

  • 单级：集合元素 （如："a", "bl"）
  • 两级：一级标签 > 二级标签 （如："bl > a"）
  • 三级：一级标签 > 二级标签 > 三级标签 （如："bl > a > t"）

排序方式：

  • 分组按集合中元素正序或逆序（有逆序标志时）排列。
  • 逆序标志仅影响该级别分组。
  • "其他"分组始终排在各组最后。
  • 组内按字母正序排列。

5. 规则示例

5.1 集合运算示例

/* 局部集合定义 */
#特殊辅音组合
BL=={bl,br,cl,cr,dr,fl,fr,gl,gr,pl,pr,sc,sk,sl,sm,sn,sp,st,sw,tr,tw}
ShortV==V>>{u}  // 短元音（去掉u）
:\b(BL)(ShortV)
@(BL)

5.2 基础规则示例

/* 位置标志 */
#元音开头
//匹配以元音开头的单词
:\bV
@
/* 完全匹配、字母逆序 */
#CVC
//匹配辅音-元音-辅音模式的三字母单词
:\bCVC\e
@-
/* 字面字符、元音分组 */
#CVCE
//匹配辅音-元音-辅音-e模式的四字母单词
:\bCVC"e"\e
@V*

5.3 组合规则示例

/* 组合规则：CVC或CVCE */
#CVC或CVCE
//匹配CVC模式或CVCE模式的单词
::CVC||CVCE
@

5.4 匹配排序示例

/* 前缀匹配 */
#元音开头
//按开头元音分组
:\bV
@V^
/* 后缀匹配 */
#辅音结尾
//按结尾辅音分组
:C\e
@C$
/* 任意位置匹配 */
#包含元音
//按包含的元音分组
:V
@V*
/* 严格中间匹配 */
#中间元音
//按中间元音分组（首尾不是元音）
:\bCV+C\e
@V~

5.5 多级排序示例

/* 逆序排序示例 */
#逆序多级
//第一级逆序，第二级正序
:\b(BL)V
@(-BL^)(V*)
/* 二级排序：辅音群开头 + 包含元音 */
#复杂音韵结构
//先按辅音群开头分组，再按包含元音分组
:\b(BL)
@(BL^)(V*)
/* 三级排序：开头 + 中间 + 结尾 */
#完整音韵分析
//按开头辅音群、中间元音、结尾辅音三级分组
:\b(BL)VC
@(BL^)(V*)(C$)

6. 注意事项与常见错误

6.1 常见错误

6.1.1 集合引用错误：

  • ❌ 错误：:\bvc\e （集合名大小写错误）
  • ✅ 正确：:\bCVC\e （集合名必须大写）

6.1.2 多字母集合引用错误：

  • ❌ 错误：:BLV （多字母集合未加括号）
  • ✅ 正确：:(BL)V （多字母集合需要括号）

6.1.3 局部集合作用域错误：

  • ❌ 错误：在规则A中定义的局部集合在规则B中使用。
  • ✅ 正确：每个规则中单独定义所需的局部集合。

6.2 使用建议

6.2.1 命名规范：

  • 使用有意义的集合名称。（如：Vowels而不是X）
  • 保持命名一致性。

6.2.2 规则组织：

  • 复杂规则建议拆分为多个简单规则。
  • 使用组合规则连接多个基础规则。

6.2.3 性能考虑：

  • 避免过度复杂的集合运算。
  • 合理使用位置标志、位置匹配符。

6.3 调试技巧

6.3.1 测试规则：使用"测试规则"功能验证规则正确性。

6.3.2 分步调试：复杂规则可以分步测试各个组成部分。

6.3.3 查看匹配结果：通过实际匹配结果验证规则逻辑。
